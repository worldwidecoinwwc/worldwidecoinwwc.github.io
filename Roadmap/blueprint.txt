1. WorldWideCoin Core Architecture
  1.Block Layer – structure of blocks
  2.Transaction Layer – transfers between wallets
  3.Consensus Layer – Proof-of-Work mining
  4.Difficulty Layer – keeps block time stable
  5.Networking Layer – communication between nodes
  6.Initially you will run single node on your PC, then later add networking.

2. Block Structure
Block
 ├── index
 ├── timestamp
 ├── transactions
 ├── previous_hash
 ├── nonce
 ├── difficulty
 ├── merkle_root
 └── hash

| Field         | Purpose                |
| ------------- | ---------------------- |
| index         | block number           |
| timestamp     | creation time          |
| transactions  | list of transactions   |
| previous_hash | link to previous block |
| nonce         | mining variable        |
| difficulty    | PoW difficulty         |
| merkle_root   | transaction hash tree  |
| hash          | final block hash       |

3. Transaction Structure
Transaction
 ├── sender
 ├── receiver
 ├── amount
 ├── timestamp
 └── signature
Later you will add cryptographic signatures to verify ownership.

4. Genesis Block
The first block must be hardcoded.
Example message:
"WorldWideCoin Genesis Block – Fair Mining for the World"

Genesis block parameters:
index = 0
previous_hash = "0"
reward = 1 WWC
difficulty = initial difficulty

5. Mining Algorithm
For now use SHA-256 Proof-of-Work.
Mining rule:
  hash(block) must start with N zeros
Example difficulty:
  difficulty = 4
Target example:
  0000a3b2c1...

6. Block Reward System
Initial reward:
  1.0 WWC
Emission model:
  reward = R0 × e^(-k × years)
Where:
  R0 = 1.0
  k = 0.025
  This gives 2.5% annual decay.

7. Difficulty Adjustment
Goal:
  1 block every 60 seconds
Adjustment every 100 blocks.
Formula:
  new_difficulty = old_difficulty × (actual_time / expected_time)
Where:
  expected_time = 100 × 60 seconds

7. Next Major Upgrades (Important)

To move closer to a real cryptocurrency like Bitcoin/Monero, the next engineering steps are:
1️⃣ Merkle tree for transactions
2️⃣ UTXO transaction model
3️⃣ P2P socket networking (not HTTP)
4️⃣ Genesis block design
5️⃣ Block validation rules
6️⃣ Mining software
7️⃣ Block explorer

UTXO (Unspent Transaction Output) model
    This is the core transaction system used by Bitcoin and many Proof-of-Work blockchains.
    Right now your system uses a simple balance scan, which works for testing but has problems:
    possible double spending
    inefficient balance calculation
    unrealistic blockchain behavior
    UTXO solves these.
  How UTXO Works
    Instead of storing balances, the blockchain stores coins as outputs.
